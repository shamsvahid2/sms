
# PRD — AMP & Forecast Dashboard  
نسخه: 1.0  
ماژول: AMP & Forecast (برنامه نگهداری و پیش‌بینی سررسیدها)

---

# 1. مقدمه (Introduction)

ماژول **AMP & Forecast** بخشی از سامانه نگهداری هوانوردی است که وظایف دوره‌ای و برنامه‌ریزی‌شده مربوط به هر هواپیما را از قبل مشخص می‌کند و زمان دقیق انجام آنها را پیش‌بینی می‌نماید.  
AMP مخفف **Aircraft Maintenance Program** است؛ یعنی «برنامه نگهداری تأییدشده برای هر هواپیما».  
Forecast نیز پیش‌بینی می‌کند **هر وظیفه چه زمانی Due می‌شود**.

این داشبورد به زبان ساده به تیم فنی کمک می‌کند بداند:  
چه کارهایی باید در چه تاریخی روی هر هواپیما انجام شود و کدام‌ها در حال نزدیک‌شدن به سررسید هستند.

---

# 2. هدف (Purpose)

هدف از این داشبورد عبارت است از:

### 1. ارائه دید کامل از وظایف نگهداری آینده  
نمایش وظایف برنامه‌ای (AMP Tasks) برای هر هواپیما با زمان باقی‌مانده تا Due.

### 2. هشدار و جلوگیری از کارهای عقب‌افتاده  
اگر یک وظیفه Maintenance در موعد تعیین‌شده اجرا نشود، باید با هشدار رنگی مشخص شود.

### 3. کمک به برنامه‌ریزی توقف‌های نگهداری  
Planner بتواند تصمیم بگیرد چه روزی هواپیما Ground شود تا کارهای برنامه‌ای انجام شود.

### 4. کمک به کاهش تعمیرات غیرمنتظره  
پیش‌بینی درست باعث می‌شود قطعات یا وظایف قبل از خرابی شناسایی شوند.

# 3. کاربران اصلی (User Types)

| نقش | نیاز اصلی |
|------|-----------|
| **Maintenance Planner** | برنامه‌ریزی انجام AMP، ساخت Work Package |
| **Maintenance Engineer** | دیدن وظایف Due برای آماده‌سازی کار |
| **CAMO Staff** | پایش تطبیق با AMP و جلوگیری از Overdue |
| **Quality Manager** | بررسی صحت برنامه نگهداری و انحراف‌ها |

# 4. داده‌هایی که باید نمایش داده شود (UI Data Requirements)

## 4.1. لیست AMP Tasks  
برای هر وظیفه نمایش داده شود:

| فیلد | توضیح |
|------|--------|
| Task Code | کد وظیفه |
| Description | توضیح وظیفه |
| Interval | بازه تکرار (FH/FC/Calendar) |
| Last Done | آخرین اجرای وظیفه |
| Next Due | تاریخ یا مقدار در آینده |
| Remaining FH/FC/Days | مقدار باقیمانده تا Due |
| Status | Green / Yellow / Red |

---

## 4.2. بخش Forecast (پیش‌بینی)  
این بخش باید نمایش دهد:

- تاریخ دقیق Due برای هر وظیفه  
- پیش‌بینی بر اساس داده‌های مصرف (FH/FC)  
- نمودار Trend برای مصرف FH/FC  
- جدول وظایف Due در 7، 15 و 30 روز آینده  
- میزان بار کاری در هفته‌های آتی (Workload Estimate)

---

## 4.3. نمودارها (Visualizations)

### 1. **Timeline / Gantt View**  
نمایش وظایف بر اساس تاریخ Due روی خط زمان.

### 2. **Remaining Life Bars**  
نوارهای رنگی برای:

- Remaining FH  
- Remaining FC  
- Remaining Days  

### 3. **Risk Color Coding**  
- سبز → مشکلی نیست  
- زرد → کمتر از 15٪ زمان باقی مانده  
- قرمز → Overdue یا کمتر از 5٪  

---

## 4.4. جدول کارهای نزدیک سررسید (Upcoming Due Tasks)

| ستون | توضیح |
|-------|--------|
| Aircraft | مربوط به کدام هواپیماست |
| Task | شرح وظیفه |
| Due In | چند FH/FC/Day مانده |
| Severity | میزان هشدار |
| Action | ساخت Work Package |

---

# 5. ورودی‌های کاربر (User Inputs)

| ورودی | کاربرد |
|--------|---------|
| انتخاب هواپیما | فیلتر AMP برای یک هواپیما |
| انتخاب کل ناوگان | مشاهده وظایف کل fleet |
| انتخاب بازه زمانی | انتخاب "7 روز آینده" یا "30 روز آینده" |
| فیلتر نوع Task | FH-Based / FC-Based / Calendar-Based |
| Sort | بر اساس کمترین Remaining |
| دکمه "Generate Work Package" | ساخت بسته تعمیراتی برای گروهی از Tasks |
| دکمه "View Task History" | نمایش سوابق اجرای یک Task |
| دکمه‌های هشدار | برای مشاهده وظایف زرد و قرمز |
